SQL será el lenguaje mediante el cual, se realizará todo tipo de transacciones ACID, a través de las peticiones SQL, estas son mis documentaciones, sobre lo elemental a saber del uso de este lenguaje, tan importante a la hora de generar conexiones a bases de datos relacionales.

Esta breve documentación, busca además exponer mis propios conocimientos y mi capacidad de explicarlos.

Los ejemplos de peticiones SQL, mostradas a continuación, han de ejecutarse en algún contexto de ejecución de las mismas, como por ejemplo "PhpMyAdmin", hecha esta aclaración comencemos. Para comentar código SQL lo hacemos con (- -), es decir doble guion.
Crea una nueva base de datos con el nombre que se le asigne, primero las palabras clave "create database" seguido del nombre de la base de datos.

- create database porfolio

Para mostrar todas las databases existentes, ejecutamos:

- show databases

Ese código nos devolverá las bases de datos existentes en nuestro admin, dentro de cada base de datos, vamos a poder declarar nuestras tablas, las cuales contendrán la información de nuestra aplicación.
Ahora utilicemos nuestra base de datos recién creada, para hacerlo usamos la palabra clave, "use" seguida del nombre de la base de datos a utilizar.

- use porfolio

De esa forma nos posicionamos en la base de datos a utilizar, ahora podemos crear una tabla en su interior, la cual tendrá usuarios dentro.

Es importante tener claro como declarar una tabla, al hacerlo debemos declarar cada uno de los campos de información, además de declarar cual es el tipo de dato que se alojara allí.

- int: simboliza números enteros.
- float: simboliza números decimales.
- varchar: es una cadena de texto, palabras, oración, frases y el valor que se le otorga es la cantidad de caracteres. 


         
   CREATE TABLE users (
            id int, 
            nombre varchar(255),
            apellido varchar(255),
            edad int,
            genero varchar(255),
            PRIMARY KEY (id)
                      );

Ahora que nuestra tabla ya está creada, veremos cono introducir información dentro de la misma, insertaremos algunos usuarios dentro, con la palabra clave "insert", de la siguiente forma: 


         
                  
   INSERT INTO users (nombre,apellido,edad,genero) VALUES('Federico','Margarito',34,'Masculino');

   INSERT INTO users (nombre,apellido,edad,genero) VALUES('Matias','Benitez',34,'Masculino');

   INSERT INTO users (nombre,apellido,edad,genero) VALUES('Macarena','Martinez',34,'Femenino');

Aquí primero declaramos el nombre de la tabla, seguido de los campos, y por último los valores que sedeamos insertar allí.


Por otro lado la palabra clave "SELECT", nos va a permitir seleccionar elementos de la db/tablas 



         
                  
         SELECT * FROM users;

Obtendríamos una tabla donde se verían los tres usuarios insertados anteriormente. El asterisco hace referencia a que acaba de seleccionar todos los elementos de la tabla "users".



       
                
         SELECT * FROM users WHERE id = 1;

Aquí solicitamos que seleccione todos los elementos donde(where) el id sea 1. En este caso debería devolvernos un solo elemento, ya que se supone que el id es único, y sería el elemento de la tabla cuyo id sea 1.



       
                
         SELECT * FROM users WHERE genero = 'Masculino';

Similar al ejemplo anterior pero esta vez, mostrara solo los elementos que tengan género masculino, aquí en cambio puede ser que más de un elemento sea masculino.



       
                
        SELECT * FROM users WHERE genero = 'Masculino' ADN nombre = 'Matias';

Podemos incorporar operadores lógicos como el AND, y así agregar un nuevo filtro que nos permite tomar más datos en cuenta. Esto se usa principalmente para ver, es decir, visualizar lo elementos, obtenerlos... a continuación, como modificar elementos ya existentes.


               

        UPDATE users SET genero = 'Femenino' WHERE id = 1;

ahora vemos que pasa si queremos eliminar un registro...




              
        DELETE FROM users WHERE id = 1;

Es importante hacer la operación de eliminación utilizando el id del elemento. Tanto en los UPDATES y en los DELETE hay que pasarle en el WHERE el id, sino tendremos un error 1175.

Ahora a profundizar la instrucción SELECT: 

 
          
            
    SELECT * from userPrototypeOne;   
    ---- de esta forma seleccionamos todos los elementos de una tabla.
        
    SELECT * from userPrototypeOne limit 1;  
    --- se limita a mostrar tan solo el primer registro.

    SELECT * from userPrototypeOne WHERE edad > 15; 
     --- de esta forma solo mostrar lo registros de esta tabla que tengan una edad superior a 15 años.

    SELECT * from userPrototypeOne WHERE edad >= 15; 
     --- aqui sera similar al anterior, sin embargo el ser mayor o igual los resultados seran otros   

    SELECT * from userPrototypeOne WHERE edad > 20 AND email = 'nico@gmail.com';   
    --- aca va a ser muy especifico y nos traera solo el registro que tenga esas caracteristicas 

    SELECT * from userPrototypeOne WHERE edad > 20 OR email = 'layla@gmail.com';   
     ---  aqui en cambio va a mostrar tanto los perfiles que tengas una edad mayor a 20 pero ademas al perfil que tenga ese email

    SELECT * from userPrototypeOne WHERE email != 'layla@gmail.com'; 
    --- nos devolvera todos los elementos que no tengan ese email

    SELECT * from userPrototypeOne WHERE edad between 15 and 30; 
    --- nos devolvera los elementos cuya edad ronde entre los 15 y 30.

    SELECT * from userPrototypeOne WHERE email like '%gmail%'; 
     --- realiza una buusqueda dentro del campo de email, pero da igual su inicio y su final, sino que realiza una busqueda de lo indicado

    SELECT * from userPrototypeOne WHERE email like '%gmail'; 
    --- en este caso, no importa como empieza la cadena, sino que lo que importa es que termine con gmail

    SELECT * from userPrototypeOne WHERE email like 'oscar%';  
    --- y aqui en cambio es al reves, le importa que empiece con oscar, pero no importa como termine la informacion.

Además de todas estas características, tenemos algoritmos de ordenamiento, en este caso, por ejemplo, podemos ordenar las edades, de mayor a menor y también de menor a mayor. 

        
 SELECT * FROM userPrototypeOne order by edad WHERE asc
 SELECT * FROM userPrototypeOne order by edad WHERE desc

Y aquí nos devolverá, el primero el mayor valor, y el segundo el menor valor de la propiedad numérica señalada, dentro de la tabla. 

      
 SELECT * max(edad) as mayor from userPrototypeOne;
 SELECT * min(edad) as menor from userPrototypeOne;

Hasta ahora el select solo lo use con el *, el cual implica que todos los elementos de la tabla están a nuestra disposición, pero podemos elegir individualmente. 


      
 SELECT id, name from userPrototypeOne;
 SELECT id, name as nombre from userPrototypeOne;

Así nos permitirá leer las columnas que solicitemos, y el segundo ejemplo nos permitirá cambiar el nombre de una columna.


Ahora veremos cómo a través de una petición, entrecruzar la información de dos tablas, a través de algo llamado llave foránea(foreign key), esta propiedad es una especie de id, que hace referencia a la información de una tabla. Dentro de otra tabla, veremos un ejemplo, en el que relacionare, la tabla de usuarios que cree userPrototypeOne, y la relacionare a una tabla de productos. 


      
    CREATE TABLE products (
      id int not null auto_increment,
      name varchar(50) not null,
      created_by int not null,
      marca varvhar(50),
      primary key(id),
      foreign key(created_by) references userPrototypeOne(id),
                                         );

como dato extra, podemos cambiar el nombre de la tabla con:



- rename table products to product;



de esa forma pasa a ser de products a product...hora veamos una nueva alternativa de insertar información en la tabla, que nos permitirá agregar más de un elemento de la tabla a la vez. 

      
   INSERT into product (name, created_by, marca)
     values
        ('ipad', 1, 'apple'),
        ('iphone', 1, 'apple'),
        ('watch', 2, 'apple'),
        ('macbook', 1, 'apple),
        ('imac', 3, 'apple'),
        ('ipad mini', 2, 'apple');

Una vez instanciada la inserción de los componentes de la tabla, veremos cómo ligar la primary key de la tabla userPrototypeOne, con la foreign key de la product.


Debemos generar dos alias dentro de esa declaración que simbolicen cada una de las tablas que van a ligarse, la tabla userPrototypeOne va a tener el alias uPO, y la tabla product será p. 

      
SELECT uPO.id, uPO.email, p.name from userPrototypeOne uPO left join product p on uPO.id = p.created_by;

además del left join, también podemos realizar un right join. 


   
SELECT uPO.id, uPO.email, p.name from userPrototypeOne uPO left join product p on uPO.id = p.created_by;

La diferencia entre left join y right join, es que uno toma como referencia la tabla de userPrototypeOne y la segunda tabla como referencia la tabla de product... en el left join podemos ver los usuarios que había creado, junto con cada producto creado, pero además teníamos a los usuarios que no habían hecho ningún producto, en cambio en el right join, es ultima información no es obtenida, ya que toma a la tabla product como punto de referencia, no mostrara elementos de la tabla userPrototypeOne que no hayan creado productos.


Por último, utilizaremos el cross join, el cual nos devuelve el producto cartesiano entre ambas tablas, es decir, toma cada elemento por individual de una de las tablas, y ese elemento lo junta con cada uno de los elementos de la otra tabla, y repite el proceso con cada elemento de la primera tabla. 


      
  SELECT uPO.id, uPO.name, uPO.edad, p.id, p.name, p.marca from userPrototypeOne uPO cross join product p;

Hay un método que nos va a permitir agrupar lo elementos según sus características... 


      
  SELECT count(id), marca from product group by marca;       
  --- nos va devolver  que hay 6 prductos de apple...

ahora bien, hagamos esto, pero ahora aplicando el left join... 


           
SELECT count(p.id), uPO.name from product p left join userPrototypeOne uPO on uPO.id = p.created_by group by p.created_by;

De esta forma tomaremos el siguiente resultado, la tabla que recibiremos nos dirá cuantos productos creo cada usuario de la tabla userPrototypeOne, seria... oscar 3, layla 2, Nicolas 1...


Ahora veremos cómo haríamos si quisiéramos excluir a Nicolas, lo haremos utilizando la propiedad having. 

        
    SELECT count(p.id), uPO.name from product p left join userPrototypeOne uPO
    on uPO.id = p.created_by group by p.created_by
    having count (p.id) >= 2;
